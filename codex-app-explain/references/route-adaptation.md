# Route Adaptation

Use this reference when the first explanation route did not resolve the user's
blocker.

## Route-change trigger

- Change route after two blocked rounds on the same concept.
- Do not repeat the same explanation shape with only superficial wording
  changes.

## Valid route changes

- analogy grounded in the user's background or everyday experience
- counterexample
- concrete-to-abstract rebuild
- fresh minimal numeric example
- coordinate or frame reinterpretation
- object-binding reset

## Failure handling

- If the source is missing, state what is missing and make the smallest safe
  assumption.
- If sources conflict, state the priority source before explaining.
- If the material does not support a rigorous explanation, stop at the
  supported boundary.

## Handoff rule

- Keep persistent memory, profile, and preference updates in the host
  environment rather than inside this skill.
